# Planning artifacts

This directory is the bootstrap planning surface for `sea_state_analyzer`.

## Purpose

Until the repo has a fuller `spec/` and ADR set, planning is intentionally lightweight:

- `in-progress/roadmap.md` holds the current roadmap and likely next slices,
- additional slice or wave documents can be added later if the work becomes large enough to benefit from them.

## Rules

- Keep planning files honest to the current repo state.
- Prefer small, verifiable slices over large speculative plans.
- Update planning when a review report changes priorities in a material way.
- Do not use planning docs to contradict the current code/test contract.
