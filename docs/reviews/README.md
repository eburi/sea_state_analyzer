# Review reports

This directory stores append-only review artifacts for this repository.

## Rules

- One review run, one file.
- Do not overwrite old reports; create a new dated report instead.
- Use the categories `HIGH`, `MEDIUM`, `LOW`, and `INFO`.
- Include reviewed scope, inputs, findings, and at least a brief negative-finding section (`reviewed without concrete finding`).
- For non-trivial work, read the latest relevant report before implementing changes.

## Naming

Use `YYYY-MM-DD-short-title.md`, for example:

- `2026-06-19-initial-code-review.md`
- `2026-07-02-release-workflow-review.md`
